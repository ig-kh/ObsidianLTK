При работе с докер образами облачных сервисов для ML/DE и проч. возможна ситуация со спец. настройкой групп пользователей - например основные пакеты ставятся `root`'ом, а работа далее под `jovyan`'ом юпитерлаба; сама работа при этом контейниризована и инстанс может подниматься под каждую рабочую сессию, соответственно переменные среды нужно поднимать каждый раз через `/etc/.bashrc`

При поддержке нескольких оболочек образом для комфортной работы ДС нужно все логин файлы в образе инкриментировать переменными:
{`export y=x:y`} $\rightarrow$ envars.txt (в часности для `$PATH` и `$LD_LIBRARY_PATH`)

```bash
echo envars.txt >> /etc/profile # sh
echo envars.txt >> /etc/bash.bashrc # bash
echo envars.txt >> /etc/zshrc # zsh
```
$\uparrow$ для интерактивных терминалов оболочек - для запусков скриптов в ремоуте это - ключевой сценарий
